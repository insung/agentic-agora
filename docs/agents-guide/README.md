# AI 에이전트 가이드 (Agents Guide)

이 디렉토리는 본 프레임워크에서 활용하는 **AI 에이전트 도구**들의 공식 명세와 주요 아키텍처 개념을 정리한 곳입니다.

## 🧠 에이전트 및 아키텍처 핵심 개념

현대의 AI 워크플로우를 고도화하기 위해 에이전트 플랫폼에서 공통적으로 사용하는 핵심 개념들입니다.

1. **MCP (Model Context Protocol)**:
   - 에이전트가 외부 데이터 소스(DB, API 등)나 도구와 통신하기 위한 개방형 표준 프로토콜입니다.
2. **SKILLS (스킬 시스템)**:
   - 에이전트의 지식과 능력을 확장하는 재사용 가능한 패키지 단위입니다. 주로 `SKILL.md` 파일을 포함한 폴더 구조로 관리됩니다.
3. **WORKFLOWS (워크플로우)**:
   - 특정 절차나 단계를 자동화하기 위해 미리 정의된 프롬프트나 실행 시퀀스입니다. 슬래시 명령어 등으로 호출합니다.
4. **멀티에이전트 아키텍처 (Multi-agent Architecture)**:
   - **서브에이전트(Subagent)**: 단일 태스크를 독립된 컨텍스트에서 수행하는 하위 에이전트입니다.
   - **에이전트 팀(Agent Team)**: 여러 에이전트가 자율적으로 협력하고 메시지를 공유하는 구조입니다.

---

## 🏗️ 에이전트 인식 원리: 프론트매터(Frontmatter)

AI 에이전트 도구들은 마크다운 파일 상단의 **YAML 프론트매터**를 통해 각 요소를 식별합니다.

1. **인식(Discovery)**: 파일 최상단에 `---` 블록(YAML)이 존재해야 시스템이 이를 에이전트나 스킬로 인식합니다.
2. **식별(`name`)**: 프론트매터 내의 `name` 필드는 시스템 내에서 해당 요소를 호출하거나 참조하는 유일한 키가 됩니다.
